EXPERIMENT · LLM-ONLY vs. DETERMINISTIC PIPELINE

Is the deterministic layer worth it?

You asked whether the post-LLM deterministic steps actually earn their keep, or whether a plain LLM call would do. I scored a raw one-shot LLM response against the full pipeline on the two golden documents. The reproducible recorded run reaches 100% after the deterministic layer; a live supporting run also clears the 90% target.

How this was measured (and why it's honest)

The keyless, reproducible baseline is the real recorded GPT response captured for each PDF. Both sides are scored against the same eval-only golden targets; golden values are never fed into either extractor. A live run on 2026-06-01 is included as supporting evidence, but recorded mode remains the reviewer gate because live outputs can vary.

LLM only

Raw cassette metrics, scored as-is. No template completion, no identity resolution, no capital-return enrichment, no unit/scale normalization, no validation.

recorded_responses/*.json → score_draft

Full pipeline

The same LLM metrics run through the complete deterministic chain in pipeline.extract(), then scored identically.

The Citi transcript states figures in billions ("$21.6 billion of revenues"), so the raw LLM emitted value=21.6, unit="billion". The template expects USD millions — so the raw numbers are off by 1000×. normalize_metrics converts billions→millions, landing all three exactly on target (difference 0).

Why this is the right division of labor

The single LLM call is good at the hard, fuzzy part — reading the document and finding the right number with a source quote. It's unreliable at the boring, high-stakes parts: getting the scale right (billions vs. millions is a 1000× error, not a rounding error) and filling fields the document only states implicitly. Those are exactly what a deterministic function does perfectly, repeatably, with no token cost and no variance.

The clearest case: a plain LLM answer would have written revenue as "21.6" into a millions column — a silent 1000× error that looks plausible and would slip past casual review. The normalizer makes that class of error impossible.
Caveat, for honesty: this scores the LLM's structured output as-is. A prompt that explicitly demanded "USD millions" might recover some scale cases — but that pushes correctness into prompt wording (high variance, model-dependent) instead of a tested deterministic guarantee. The point of the deterministic layer is to make these guarantees not depend on prompt luck.
