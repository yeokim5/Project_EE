Part 1 — Reliability: works on any document & company

An unseen-company smoke test (Netflix) failed with "No supported earnings PDFs found". Root cause wasn't the LLM — it was three deterministic gaps. Each fixed with a narrow, tested heuristic. No company-specific tokens, no full LLM rewrite (which would have broken recorded mode, the keyless eval gate, and the sub-$5/PDF cost story).

📄 Whole-document scan

The classifier only checked the first 6 pages for financial statements, so back-loaded shareholder letters (Netflix: statements on pages 9, 11–14) were dropped as unknown. Now it scans all pages.

classify.py · has_statement → full_text

🎙️ Structural transcript gate

A lone mention of "earnings call" could flip a report into a transcript. Now it requires structure: speaker turns + an operator: line, or the phrase backed by ≥2 speaker turns. Citi still classifies right via its real structure.

classify.py · has_transcript_structure

🔢 Thousands scale

Smaller filers report "$ in thousands"; those values came out 1000× too large. Normalization now divides the thousands/k scale down to the canonical USD-millions basis.

normalize.py · thousands / k
Result: Netflix and any back-loaded letter now classify, select the right statement pages, and extract. Covered by new tests in tests/test_classify.py and tests/test_phase4_processing.py. Tesla and Citi eval stay 9/9.

Part 2 — Pixel-accurate citations

Every number already carried a source page + quote, machine-verified to be present on that page. This adds the visual layer: the exact rectangle on the page, derived deterministically — the LLM never produces coordinates.

What was added

FileWhat it is
earnings_extractor/locate.pylocate_evidence_bbox(pdf, page, quote) → rects + page size. PyMuPDF search_for with chunked + numeric-anchor fallbacks for quotes that wrap or have odd whitespace. Additive — no change to extraction, schema, or recorded mode.
scripts/build_citation_viewer.pyGenerates a self-contained HTML per draft (PDF embedded, PDF.js from CDN). Read-only viewer over a finished draft.
tests/test_locate.py4 tests: known quote → rect on correct page; wrapped statement line resolves; absent quote → no rects; out-of-range page → empty.
On real data: 11/12 Tesla citations and 9/9 Citi citations resolved to highlights. The one Tesla miss is the intentionally-blank buyback placeholder — correctly badged "no on-page match" rather than faked.

Build a viewer yourself

# recorded mode — no API key needed
python -m earnings_extractor extract assesment_info/citi_earnings_q12025.pdf --out outputs/test_citi --mode recorded
python scripts/build_citation_viewer.py \
  --draft outputs/test_citi/draft_metrics.json \
  --pdf assesment_info/citi_earnings_q12025.pdf \
  --out outputs/citations_citi.html
# open outputs/citations_citi.html → click a metric → highlight lands on its number
Scope honesty: this is the read-only viewer slice of Phase 10 — the reusable foundation. The full interactive app (browser upload, live per-stage pipeline, in-UI review + export, FastAPI/Streamlit backend) is still post-submission. The viewer renders a finished draft; it does not upload, extract, or export.
