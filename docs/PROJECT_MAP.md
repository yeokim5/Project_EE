## How This Stays Current

- The tree block between `FILE_TREE` markers is generated. Do not hand-edit that
  block.
- Everything outside the markers is hand-written context. Keep it concise so it
  stays useful.
- Generated runtime outputs belong under `outputs/` and are intentionally
  excluded from the tree.
- Local-only environment and cache files are excluded from the tree.

## Current Top-Level Files

- `.env.example` - documents required environment variables without storing
  secrets.
- `.gitignore` - keeps secrets, caches, virtualenvs, and generated outputs out
  of git.
- `AGENTS.md` - durable rules for coding agents working in this repo. Do not
  auto-edit it for one-off notes.

## Current Directories

- `assesment_info/` - raw assignment inputs from the client brief: PDFs, email
  context, and the sample Excel template. Treat these as source materials.
- `docs/` - planning, requirements, eval rules, workflow notes, verification
  gates, and decision records.
- `scripts/` - small local maintenance tools that support development but are
  not part of the runtime extraction package.

## Planned Runtime Shape

- `earnings_extractor/` - deterministic review-first extraction package. This
  will hold CLI commands, PDF ingestion, LLM extraction adapters, normalization,
  validation, review artifact generation, and export code.
- `tests/` - unit and integration tests for schema validation, normalization,
  export formatting, review decisions, and CLI behavior.
- `evaluation/` - eval-only golden fixtures and tolerance rules. Golden
  expected values must never be importable by extractor code.
- `tests/fixtures/` - recorded-response fixtures when the LLM cassette path is
  added.

## Generated Artifacts

The normal draft path should produce these under `outputs/<run_id>/`:

- `draft_metrics.json`
- `review_queue.json`
- `evidence_report.md`
- `review.html`
- `review_decisions.json`

The reviewed export path should produce:

- `outputs/extractions.xlsx`
- `outputs/extractions.json`
- `outputs/audit_report.md`

These files are useful for verification, but they stay out of git unless a
specific fixture is intentionally promoted into a test directory.
