Earnings Extractor

Review-first extraction of standard financial metrics from earnings PDFs and call transcripts into a client-ready Excel workbook, with citation review and a full audit trail.

LLM draft → deterministic validation → citation review → approved Excel export
Live Demo Source Code

What it does

This is not just “an LLM extracts numbers.” The model only does the fuzzy reading — find each metric, quote its source sentence, report a confidence. Everything that has to be reliable runs as fixed, testable code, and numbers reach the client workbook only after a human approves the citation behind them.

Given an earnings report or earnings-call transcript, the tool produces a draft dataset of standard metrics (revenue, net income, EPS, operating income, gross margin, operating expenses, buybacks & dividends), validates every value against its own source evidence, flags anything uncertain or unsupported for human review, and exports an Excel workbook that matches the client template — plus JSON and a Markdown audit report.

Architecture & data flow

flowchart TD A[PDF input] --> B[Ingest] B --> C[Classify document & pages] C --> D[Structured LLM extraction
value + quote + confidence] D --> E[Normalize units & scale
to USD millions] E --> F[Validate
evidence + consistency + magnitude] F --> G[Confidence / review flags
triage, not proof] G --> H[Human review
approve / reject / note] H --> I[Final export
Excel + JSON + audit]

The orchestration is fixed code, not an agent. The LLM is one constrained, schema-bound call; ordering and control flow never depend on the model.

On the reproducible recorded A/B, the same raw model outputs move from 15/18 to 18/18 after normalization, enrichment, and validation. Golden values live only in evaluation/; runtime code never imports them, and a test enforces that separation.

Confidence is review triage, not proof

A high model confidence never lets a value skip validation, and validation never edits a value — it only marks it for review with a reason. A value is routed to a human when any of these is true:

  • confidence is below the review threshold (0.75);
  • the source quote is missing;
  • the quote is not found on the cited page;
  • the extracted number is not grounded in its quote;
  • a required template field is blank;
  • the field is unsupported for the document type;
  • a consistency check fails (OCF − capex must reconcile to free cash flow);
  • a magnitude check fails (net/operating income vs. revenue; gross margin range).

Understanding the Excel workbook

SheetContents
Client TemplateFirst sheet, exact client columns & formatting. Only approved values appear.
MetricsNormalized extracted values plus each field's review decision.
Review DecisionsApprove / reject / not-applicable / note record per metric.
EvidenceSource page, quote, confidence, and review-flag reason per metric.

Formatting: revenue / net income / operating income / operating expenses render as $<n>B (stored as USD millions); gross margin as a percentage cell; EPS as a plain number; unsupported or unapproved values stay blank — never guessed. Demo/unreviewed exports are clearly marked DRAFT/UNREVIEWED.

Run it

Setup

python -m venv .venv
source .venv/bin/activate
python -m pip install -e ".[dev]"

CLI demo — no API key (recorded mode replays committed responses)

python -m earnings_extractor extract assesment_info --out outputs/run_001 --mode recorded
python -m earnings_extractor inspect outputs/run_001/draft_metrics.json
python -m earnings_extractor review outputs/run_001 --out outputs/run_001/review.html --demo-decisions outputs/run_001/review_decisions.json
python scripts/make_acceptance_decisions.py outputs/run_001 --out outputs/run_001/human_review_decisions.json
python -m earnings_extractor export outputs/run_001 --decisions outputs/run_001/human_review_decisions.json --out outputs/extractions.xlsx

Output artifacts

  • draft_metrics.json — normalized draft rows, evidence, validation flags, token usage.
  • review_queue.json — rows requiring a decision.
  • evidence_report.md — citation-oriented audit summary.
  • review.html — static review page: page, quote, confidence, flag reason, decision controls.
  • extractions.xlsx / .json / .audit.md — the reviewed export and its audit trail.

Limitations & production path

V1 covers earnings PDFs and call transcripts. The golden eval set has two validated companies; production should expand to 30–50+ labeled documents, add confidence calibration from historical outcomes, connect ingestion to the client's workflow, and move the local review artifact into a persistent multi-user review queue with audit logs, SSO/RBAC, and monitoring for accuracy, review rate, failure rate, and cost per document.
